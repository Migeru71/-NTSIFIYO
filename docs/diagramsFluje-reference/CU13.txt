@startuml
skinparam sequence {
    LifeLineBackgroundColor #F5F5F5
    ParticipantBackgroundColor #FFFDD0
    ParticipantBorderColor #666666
}
skinparam NoteBackgroundColor #F5F5F5

title CU-013: Crear nuevas actividades / CU-014: Asignar puntuaciones

actor "Docente" as T
boundary "ConfiguracionActividadView" as View
control "ApiService" as Api
control "ActivityController" as Ctrl
entity "ActivityService" as Srv
entity "ExperienceService" as ExpSrv
entity "GameRepository" as Repo
database "MySQL" as DB

autonumber

T -> View: Selecciona tipo de actividad y configura medios
note right: Configura para cada elemento:\n- Medios: Texto/Imagen/Audio\n- Idioma: ESP/MAZ

T -> View: Completa datos base (nombre, descripción, dificultad)
View -> Api: requestScore(configDto)
Api -> Ctrl: POST /api/activities/calculate-score

group Lógica de Puntuación (CU-014)
    Ctrl -> ExpSrv: calculateSuggestedXP(dto)
    ExpSrv -> ExpSrv: Evaluar complejidad de medios y tipo de juego
    ExpSrv --> Ctrl: recommendedXP
end

Ctrl --> Api: 200 OK (recommendedXP)
Api --> View: Muestra puntuación recomendada

T -> View: Confirma o modifica el valor de la experiencia
View -> Api: createActivity(finalDto)
Api -> Ctrl: POST /api/activities
Ctrl -> Srv: createGameActivity(finalDto)

alt #FFFDD0 Datos incompletos o palabras insuficientes
    Srv --> Ctrl: throw ActivityIncompleteException
    Ctrl --> Api: 400 Bad Request
    Api --> View: Error detectado
    View --> T: Mostrar: "No son suficientes palabras o faltan campos"
end

alt #FFFDD0 Experiencia fuera de rango o vacía
    Srv -> ExpSrv: validateXP(value, config)
    ExpSrv --> Srv: invalid
    Srv --> Ctrl: throw InvalidXPException
    Ctrl --> Api: 422 Unprocessable Entity
    Api --> View: Alerta de rango excedido
    View --> T: Mostrar: "Se usará el valor por defecto del sistema"
end

Srv -> Srv: buildGameEntity(dto)
note right: Aquí el Service construye el objeto Game\nque ya incluye su lista de Questions\no su lista de Words (Pairs).

Srv -> Repo: save(gameEntity)
Repo -> DB: INSERT INTO Game (cascading children)
DB --> Repo: confirmation
Repo --> Srv: storedGame

Srv --> Ctrl: ActivityCreatedDto
Ctrl --> Api: 201 Created
Api --> View: SuccessNotification
View --> T: Notifica creación exitosa

@enduml
